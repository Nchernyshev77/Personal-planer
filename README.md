# Planner (браузерный)

Минималистичный планировщик микро‑задач:
- добавление задач
- перетаскивание для изменения порядка
- отметка «выполнено»
- цветные метки
- PiP / поверх окон (Chrome) + fallback окно

## Сохранение прогресса

Приложение **всегда** сохраняет резервную копию в `localStorage`.

Для “настоящего” **автосейва в текстовый файл** используйте Chrome/Edge и кнопку **«Подключить tasks.txt»**:
- выбираете `tasks.txt` (или `.json`)
- после каждого изменения файл **перезаписывается**
- при следующем открытии приложение попробует восстановить доступ к файлу автоматически  
  (браузер может попросить подтвердить разрешение — это ограничение безопасности).

## GitHub Pages (без установок)

1. Создайте репозиторий и загрузите файлы:
   - `index.html`
   - `style.css`
   - `app.js`
   - `popout.html`
2. В GitHub: **Settings → Pages**
3. Source: **Deploy from a branch**
4. Branch: `main`, Folder: `/root`

Готово — открывайте ссылку GitHub Pages.

## Примечание про автосейв в файл

File System Access API требует:
- поддерживаемый браузер (обычно Chrome/Edge)
- secure context (`https` или `localhost`)

Если API недоступен, приложение продолжит работать только с `localStorage`.
